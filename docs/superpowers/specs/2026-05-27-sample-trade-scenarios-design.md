# 样本贸易场景扩展设计

## 目标

在 `init_sample_trade.py` 中新增 4 个样本贸易场景，覆盖不同贸易术语、付款方式、运输方式和产品类型。按场景特点自然调整流程和单证，不做不必要的模型改造。

## 现有场景

- **CIF + L/C**：蓝牙耳机（HS 85183000）出口美国，海运 40'HC

## 前置条件：模型改造（3 项）

### 1. Shipment 模型增加空运支持

- 新增 `transport_mode` 字段（ForeignKey to TransportMode，默认值指向海运）
- 新增 `awb_no` 字段（CharField, blank=True, null=True），存储航空运单号
- `vessel_name` 改为 blank=True（空运时为空）

### 2. LetterOfCredit 改为可选

- Contract 与 LetterOfCredit 的 OneToOneField 改为 nullable（blank=True, null=True）
- T/T 场景下不创建 L/C 记录

### 3. ForexSettlement 支持多次结汇

- 将 Transaction 与 ForexSettlement 的关系从 OneToOne 改为 ForeignKey（一对多）
- 支持 T/T 30%+70% 分批结汇

## 新场景设计

### 场景 1：FOB + T/T（电子消费品出口中东）

| 属性 | 值 |
|------|------|
| 出口商 | 深圳华芯电子科技有限公司 (EXP02) |
| 进口商 | 迪拜 Al Rashid Trading LLC (IMP02) |
| 商品 | 智能手表（HS 910212），2,000 只 |
| 单价 | FOB 深圳 USD 25.00 |
| 总价 | USD 50,000 |
| 贸易术语 | FOB Shenzhen |
| 付款方式 | T/T（30% 预付 + 70% 见提单副本） |
| 物流 | 海运 1×20'GP，深圳 → 杰贝阿里港 |

**流程调整**：
- 无信用证流程
- 无保险流程（FOB 下卖方不负责保险）
- 两次外汇结算：预付款 + 尾款
- 退税率：13%（电子产品）

**单证清单**：
1. 商业发票
2. 装箱单
3. 海运提单
4. 产地证 FORM A
5. 报检申请 / 检验证书
6. 报关单
7. 装船通知
8. 受益人证明

## 实现约束

- 传感器归入 `electronics` 分类，不新建 `precision_instruments`
- 所有单证编号互不重复（使用日期+随机编号机制）
- 日期逻辑合理（各场景时间线不重叠）
- 金额计算准确（保险 = CIF/CIP × 110%）
- 在同一个 `init_sample_trade.py` 文件中添加（方案 A）
