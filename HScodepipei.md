# HS 编码匹配与税率展示修复记录

## 问题描述

在 http://localhost:8000/market/ 商品列表页面，税率信息完全不显示。

## 根因分析

两层原因导致问题：

1. **HS 编码长度不一致**：商品表（Product）中的 HS 编码为 8 位（如 `85021100`），而 HSCode 参考表中的编码为 10 位（如 `8502110000`）。后端使用精确匹配（`hs_codes_cache.get(p.hs_code)`），导致全部匹配失败，税率字段为空。

2. **数据缺失**：6 个章节（09, 16, 22, 32, 52, 69）的 HSCode 数据完全没有同步到数据库。

## 修复内容

### 1. 代码修复：渐进式前缀匹配

文件：`apps/products/views.py`（第 97-108 行）

新增 `_find_hscode()` 函数，替代原来的精确匹配：

```python
def _find_hscode(code):
    """精确匹配，失败则渐进前缀匹配（8位→6位→4位）"""
    if code in hs_codes_cache:
        return hs_codes_cache[code]
    for prefix_len in (8, 6, 4):
        prefix = code[:prefix_len]
        if len(prefix) < prefix_len:
            continue
        for hsc_code, hsc in hs_codes_cache.items():
            if hsc_code.startswith(prefix):
                return hsc
    return None
```

匹配策略：
- 首先尝试精确匹配（10 位编码完全一致）
- 失败后尝试 8 位前缀匹配（覆盖 8 位→10 位的常见情况）
- 再失败尝试 6 位前缀匹配（HS 编码子目级别）
- 最后尝试 4 位前缀匹配（HS 编码品目级别）

### 2. 数据同步

补充同步了缺失的 6 个章节 HSCode 数据：

```bash
python manage.py sync_hs_codes --chapter 09,16,22,32,52,69 --with-tax
```

同步结果：
- 第 09 章（咖啡、茶等）：93 条
- 第 16 章（肉鱼制品）：105 条
- 第 22 章（饮料、酒）：52 条
- 第 32 章（鞣料染料）：126 条
- 第 52 章（棉花）：400 条
- 第 69 章（陶瓷产品）：数据随上述命令一同同步
