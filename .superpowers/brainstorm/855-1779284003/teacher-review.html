外贸老师的评估

从教学和业务角度评估数据模型

✅ 优点

设计合理的地方
  • 用户-角色分离 — 一个学生可以扮演多个角色，符合实训需求
  • 操作日志 — ActivityLog 支持过程评分，这是关键
  • 课程维度 — 所有数据都关联到 Course，便于教学管理
  • 单证模板化 — 教师可以自定义单证模板
  • 角色配置 — Course.role_settings 支持教师灵活配置

⚠️ 需要补充的

缺失但重要的数据结构
1. 财务/费用相关
// 学生需要计算成本、利润，体验盈亏
TransactionCost (交易费用)
id, transaction_id → Transaction
cost_type (货款/运费/保费/报关费/银行费/其他)
amount, currency, paid_at
2. 系统基础数据
// 汇率、费率由教师调整，模拟市场变化
ExchangeRate (汇率)
from_currency, to_currency, rate, effective_date
ShippingRate (运费率)
from_port, to_port, rate_per_unit
InsuranceRate (保险费率)
coverage_type, rate_premium
3. 地理数据
Country (国家)
code, name, currency, calling_code
Port (港口)
code, name, country_id → Country
4. 消息通知
// 角色间需要消息流转（如"信用证已开立"）
Message (消息)
id, from_user_id, to_user_id, course_id
type (系统/交易/单证), title, content
is_read, created_at
5. 单证关联
// 单证之间有依赖关系，如提单依赖报关单
DocumentDependency (单证依赖)
id, doc_id → Document, depends_on_doc_id
dependency_type (前置/同步/后置)
