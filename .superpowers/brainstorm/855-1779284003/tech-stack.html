技术栈方案

兼容 Win7 IE 浏览器 + Python 后端

Windows 7 IE 浏览器兼容性约束：

  • Win7 自带 IE8/IE9，用户可能升级到 IE10/IE11
  • 不支持现代前端框架（React/Vue/Angular 2+）
  • 需要 ES5 语法 + polyfill
A

经典服务端渲染

Python 模板引擎（Jinja2/Django Templates）+ jQuery

优点

  • IE 兼容性最好
  • 开发简单直接
  • SEO 友好

缺点

  • 交互体验较弱
  • 服务器负载较高
B

混合方案

服务端渲染为主 + 局部 Vue 2.x（IE11 兼容）

优点

  • 平衡兼容性与体验
  • 复杂交互用 Vue
  • 渐进式增强

缺点

  • 需要用户升级到 IE11
  • 技术栈稍复杂
C

纯 jQuery + AJAX

Django/Flask 提供 JSON API，前端完全用 jQuery

优点

  • IE8+ 完美兼容
  • 前后端分离清晰
  • 体验接近 SPA

缺点

  • 大量 DOM 操作代码
  • 状态管理复杂

Python 后端框架：

  • Django — 全功能框架，内置管理后台，适合快速开发
  • Flask — 轻量灵活，组件自由选择
  • FastAPI — 现代异步，自动 API 文档，但较新
