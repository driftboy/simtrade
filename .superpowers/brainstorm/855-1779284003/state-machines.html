工作流状态机设计

核心业务状态流转

1. 交易状态机 (Transaction)

交易完整生命周期
草稿 → 询盘中 → 还盘中 → 待签约 → 已签约 → 履约中 → 已完成
| 已取消

2. 单证状态机 (Document)

单证制作与流转
草稿 → 待审核 → 已审核 → 已提交 → 已接收 → 已归档
| 审核不通过 → 草稿(修改)

3. 信用证状态机 (LetterOfCredit)

L/C 开立与流转
草稿 → 待开证 → 已开证 → 已通知 → 待议付 → 已议付 → 已付款

4. 生产订单状态机 (ProductionOrder)

工厂生产流程
待确认 → 已确认 → 生产中 → 已完成 → 已发货

5. 付款状态机 (Payment)

资金结算流程
待支付 → 处理中 → 已支付 → 已收款
| 已取消

💾 数据库实现

状态机数据结构
方案一：状态字段 + 状态历史
# 每个业务表添加 status 字段
transaction.status = CharField(choices=STATUS_CHOICES)

# 状态历史表（追踪流转）
TransactionStatusHistory
id, transaction_id → Transaction
from_status, to_status, changed_by → User
change_reason, created_at
方案二：工作流引擎表（更灵活）
# 工作流定义
WorkflowDefinition
id, entity_type (Transaction/Document/...)
current_state, next_states (JSON), transitions (JSON)

# 状态转换规则
WorkflowTransition
id, workflow_id → WorkflowDefinition
from_state, to_state, action, required_role
condition (Python表达式), auto_transition (bool)
